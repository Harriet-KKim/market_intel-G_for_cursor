# 01. System Overview

## 목표
특정 회사 및 기술 키워드에 대해 최신 정보를 짧은 주기로 수집하고, 일정 기간 누적된 신호를 압축·해석하여 전략 분석에 활용한다.

## 핵심 설계 원칙
- **속도와 깊이를 분리**: 일간 수집과 주간 해석을 분리한다.
- **신호 중심 구조화**: 기사 단위가 아니라 이벤트 단위로 지식을 축적한다.
- **출처 기반 검증**: 원문, 공식 자료, 2차 인용을 구분한다.
- **누적형 지식화**: 단건 요약이 아니라 회사별·테마별 위키 형태로 유지한다.
- **행동 가능성 확보**: 최종 산출물은 의사결정에 쓸 수 있어야 한다.

## 시스템 레이어
### 1) Monitoring Layer
수집 대상:
- 뉴스
- SNS
- 연구자료/논문
- 특허
- 채용공고
- 공식 블로그/보도자료
- GitHub/개발 문서
- 컨퍼런스 발표 자료

주요 기능:
- 수집
- 엔터티 태깅
- 언어 감지
- 1차 요약
- 중복 제거 후보 생성

### 2) Validation Layer
주요 기능:
- 재인용 기사 제거
- 독립 출처 판정
- 사실 주장 추출
- 상충 정보 감지
- 출처 신뢰도 평가
- 중요도 부여

### 3) Knowledge Layer
주요 기능:
- 회사별 프로파일 갱신
- 일간 인텔 로그 축적
- 소스 카드 저장
- 테마 문서 연동
- 변화 이력 관리

### 4) Weekly Intelligence Layer
주요 기능:
- 주간 이벤트 클러스터링
- 핵심 변화 식별
- 산업 맥락 반영
- 경쟁사 비교
- 리스크/기회 해석
- 전략 제안 도출

### 5) Executive Layer
최종 출력:
- 이번 주 Top signals
- 회사별 주요 변화
- 시장 패턴
- 시사점
- 다음 조사 우선순위

## 기대 효과
- 최신성 유지
- 중복/노이즈 감소
- 해석 품질 향상
- 내부 축적 자산 형성
- 리서치 운영 비용 최적화
