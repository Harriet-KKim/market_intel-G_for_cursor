# 02. Operating Workflow

## 일간 운영 흐름
### Step 1. 대상 정의
- 추적 회사 목록 정의
- 회사별 핵심 기술/시장 키워드 정의
- 우선순위 회사 티어링 (Tier 1 / Tier 2 / Watchlist)

### Step 2. 자료 수집
수집 소스별 기본 역할:
- 뉴스: 외부 발표와 시장 반응 포착
- SNS: 초기 신호 탐지
- 논문/연구자료: 기술 성숙도 추적
- 특허: 방향성 탐지
- 채용공고: 실제 투자/조직 변화 탐지
- 보도자료: 공식 메시지 파악
- GitHub/문서: 개발 진행 정황 포착

### Step 3. 1차 정규화
- URL canonicalization
- 제목/본문 정리
- 언어 통일
- 회사/기술/제품/지역 엔터티 태깅
- 날짜 표준화
- 중복 문서 식별

### Step 4. 1차 모델 처리
저비용 모델 활용:
- 한 줄 요약
- 주요 주장 추출
- 관련 키워드 태깅
- 소스 유형 분류
- 후속 검증 필요 여부 플래그

### Step 5. 검증 에이전트 처리
- 독립 근거 수 확인
- 같은 주장 반복인지 확인
- 원문 출처 여부 판정
- 홍보성 표현 제거
- 불확실 정보 분리

### Step 6. 이벤트화
문서들을 아래 형태의 이벤트로 묶음:
- 이벤트명
- 관련 회사
- 관련 기술
- 핵심 주장
- 근거 소스들
- 확실도
- 중요도
- 전략적 메모

### Step 7. 저장
- 회사별 daily intel log 갱신
- source card 저장
- 관련 테마 문서 링크
- company profile 업데이트 후보 등록

## 주간 운영 흐름
### Step 1. 주간 수집분 집계
- 회사별 이벤트 묶음 생성
- 중복/저품질 이벤트 제거
- 변화량 계산

### Step 2. 고급 모델 기반 압축
고성능 모델 활용:
- 이벤트 간 관계 파악
- 방향성 변화 추론
- 경쟁사 비교
- 산업 전체 맥락 연결
- 전략 의미 해석

### Step 3. 전략 브리프 작성
포함 항목:
- 이번 주 핵심 변화
- 경쟁 환경 변화
- 기술/공급망/제품/조직 측면 시사점
- 루머와 사실 구분
- 다음 주 확인 포인트

### Step 4. 피드백 반영
- 잘못된 요약 수정
- 키워드 사전 보강
- 우선순위 회사 조정
- 에이전트 룰 조정

## 운영 리듬 예시
- 하루 3~6회 수집 배치
- 하루 1회 검증 재처리
- 주 1회 전략 압축
- 월 1회 키워드 사전 및 소스 리밸런싱
